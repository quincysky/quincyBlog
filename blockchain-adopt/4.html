    Blockchain adoption - A value driver perspective

    vuePress-theme-reco    2022

    Blockchain adoption - A value driver perspective

    # 摘要

    该论文探讨了区块链技术及其潜在价值驱动因素之间的关系。并且分析和讨论了区块链发展的四个阶段。这四个阶段的技术分别专注于交易、智能合约、去中心化应用以及支持人工智能的去中心化决策。此外，提出了一种以区块链价值驱动因素为中心的决策框架。为从业者提供了一种针对特定场景采取合适区块链技术的方法。对于学者，批判性的识别和评估了区块链阶段的特征及其战略意义，并提供了一种结构化的方法来概念化区块链技术演进。

    # 战略区块链观点

    早期采用区块链的人们已经开始将区块链技术应用于从供应链管理到医院记录、政府通过透明投票的运作以及电子身份证的数字签名等各个方面。

    # 区块链成熟阶段

    最开始的区块链还不是可编程的，直到比特币的出现。可编程区块链的概念并不一定会使其优于非可编程区块链，但是它确实实现了特定的功能，并扩大了市场去中性化的范围。

    不同种类的区块链技术和用途具有以下四个阶段的特点：(前三个阶段已经建立，第四个阶段是新兴的人工智能区块链集成)

    • 区块链1.0专注于交易，主要是在与现金相关的应用程序中部署加密货币，如货币转账、汇款、和数字支付系统。
    • 区块链2.0是区块链1.0的扩展，它包含隐私、智能合约、以及非本地资产区块链代币和功能的出现。
    • 区块链3.0纳入了去中心化应用程序。dApp具有灵活、透明、分布式、有弹性的特点，并且具有明确的激励结构。创建去中心化存储和去中心化计算可能性使dApp具有更大的可扩展性。
    • 人工智能是基于概率理论来表达不确定性。他是不断变化的，算法被用来猜测或假设现实。区块链则是用来确定性哈希算法，当输入保持不变时，该算法会产生相同的结果。尽管技术不同，但是他们的联合使用有助于解决复杂的问题。CognitiveScale是这种组合技术使用的一个成功例子。算法管理(AM)形式的自动决策包含采取管理功能的软件算法和支持算的设备。AM最好应用于可以进行预定义的稳定环境。AM可以自由地根据不良表现采取行动并实时采取纠正措施以提高质量、分配适当的工作任务等。合并实时信息也是一种选择，AM用于调整智能合约的参数和条件。如下图所示，AI做出存储在区块链中的决策。数据存储和区块链这两个概念在去公共中心化的账本概念是相同的。但在私有账本中并不是对所有都是相同的。私有账本可能在数据存储和区块链记录之间有强大的防火墙，只公开决策，而不公开他们的实际数据。区块链4.0尤其适用于高度重视可追溯性和不变形的环境。

    # 区块链价值驱动因素的基本逻辑

    纵观区块链的演变，我们可以确定某些驱动因素如何创造价值的基本逻辑。区块链1.0与交易成本的降低直接相关。区块链2.0通过支持智能合约的开发和使用，向前迈出了一步。区块链3.0引入了去中心化应用程序和计算的概念。最后区块链4.0是区块链和人工智能联合使用的结果。这使得系统能够做出决策并采取行动，而无需直接人为干预。这种通过设置参数作出判断，然后独立于管理监督执行管理功能的能力是其主要价值驱动因素。

    # 价值创造影响

    尽管各个区块链阶段是建立在彼此之上的，但每个阶段都有不同的目的。对于价值创造，给定服务所需的功能比使用的技术更加重要。尽管四个阶段的进化要素，但这并不意味着追求和实施更高阶段就一定能获得最高价值。 一个组织不必使用从区块链1.0开始，然后通过随后以串行顺序实施阶段2.0到4.0来丰富其技术应用。一下是作者给出的开发建议：

    • 实施和使用合适的区块链技术阶段比追求更高的技术水平更重要。
    • 技术阶段本身并不意味着他会产生更多的价值。技术的使用方式决定了其产生的价值而不是其功能。
    • 使用不同阶段的技术不应直接就其价值创造进行比较。

    # 针对炒作的结构化决策

    作者为管理者提供了一个结构化框架，用来评估采用和部署的区块链设计方案。接下来讨论了有助于确定合适的区块链技术的四个关键问题。如下图的一个框架。

    # 价值驱动因素和价值机会

    寻求什么样的价值？

    价值驱动因素能够是降低基于交易的成本、增加新服务、划定组织边界以及决策的自动化和去中心化。价值创造的驱动因素与通过使用区块链获得特性和功能有内在联系。

    # 可行性

    采用区块链技术是否可行？

    鉴于区块链技术的新兴性质，从头开始构建和维护区块链的应用程序很容易成为一项相当艰巨和昂贵的工作，而且针对特定应用程序的预构建解决方案的选择仍然相对较少。因此，只有当使用区块链有很强的预期收益时，应追求区块链。

    # 合适的技术选择

    为什么区块链优于中心化账本？

    即使区块链的激增，但本身并不会导致产品或业务流程的新的提升，也就意味着使用区块链不是最佳选择。

    区块链如何增肌价值而是其他技术无法做到的呢？有两个方面：

    1. 第一个涉及使用分布式而非集中式分类账本，例如ERP系统
    2. 第二个方面涉及数据所有权。

    Juniper Research报告说，在不首先考虑其他替代方案的情况下，不要使用区块链，因为在许多情况下，系统性而非技术的变革可能是一种更好更便宜的解决方案。除成本外，不必要的区块链引入可能会倒置严重的内部和外部中断。由于潜在风险，财产登记或个人身份识别实例可能不可行。区块链的51%攻击是一种可信且重大的威胁。另一个风险是，拥有被泄露的私钥的用户无法恢复它，这与通常可以重新发布的集中式系统不同。

    什么样的技术组合与追求的价值相一致？

    一旦管理层决定使用区块链，决策过程的下一步就是确定所选的技术必须拥有的特性。

    # 区块链价值驱动框架

    基于以上的四个问题，建议使用区块链价值驱动框架来捕捉技术的可行性，因为它提供了强有力的指示。

    # 更广泛的区块链应用视角

    作者建议管理者仔细考虑与区块链相关的风险，在将技术实际融入到现有的产品时，最好从小处着手。关键是确保所选技术所提供的价值最符合总体战略。
